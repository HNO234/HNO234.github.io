閱讀《原子習慣》後心得與在生活中的實踐

回歸第一周偷懶講非技術的東西

我認為的知識分享本來就沒有要限定在資訊或技術，偶而可能會有這種非技術但我認為很有感想的。

我在寒假的時候閱讀《原子習慣》。之前就聽說這本很久了，花了幾天讀完以後收穫真的滿多的，這裡就來簡單分享一下書中的內容與讀完一個月下來在生活中的實踐。

內容概述

這本書主要在描述各種習慣（包含好習慣和壞習慣）的主要建立過程，以及要怎麼利用這個機制去更好的建立好習慣與破除壞習慣。

在進入主要內容以前，他還有提到一些有關習慣一些論點，其中讓我最有印象的是以下兩點：

  • 「造就成功的，是日常習慣，而不是千載難逢的轉變」
  • 「改變習慣最有效的方法，是改變身份認同」：例如說如果你想多運動，應該認為「自己是個喜歡運動的人」，而不是「自己正在嘗試運動」。

我認為這兩點對我最大的幫助就是心態調整吧。例如說，我以前的 mindset 會比較偏向「我正在嘗試學 XXX 技術」，現在我儘量會把自己想成「我是個喜歡資訊科技的人」或「我打算以資訊當作志業」。有了這個心態以後，會自然去想像自己想成為的那種人，例如自己就想成為一個能侃侃而談的資訊教育者。每個禮拜會去檢討自己的習慣是否能讓我成為自己想成為的人，以及下個禮拜要怎麼做調整。

之後書就把建立習慣簡單的分成了四個階段，分別為提示、渴望、回應、獎賞。底下就針對我比較有印象的點來講。

提示

提示就是自己的感官察覺到什麼東西。沒有提示，習慣根本就不會開始。例如說，當你看到手機，連結了自己想看社群動態的渴望，就會把手機拿起來滑。

書中提到比較有印象的一些觀點：

  • 習慣都是不假思索的，所以如果要改善自己的習慣就要從覺察開始。可以把自己的習慣寫在一張紙上，下次做的時候就會更容易注意到。
  • 習慣的建立要有一個明確的時程規劃，例如「我要在幾點做什麼」。
  • 環境往往比自制力更重要，要建立某個習慣就把相關的東西擺在顯眼的地方，要破除某個習慣就把相關的東西藏起來。

最後一點對我的影響力應該是最大的。以拖延來說，以前常常會去譴責自己為什麼自制力不夠，但是這樣做其實沒有改變任何事，因為人性很難對抗。現在會去檢討環境是不是不夠好，例如說我現在用這個插件 把 Youtube、Discord 跟 FB 的連結轉到 GitHub，讓這些浪費時間的潛在因子下降（至少我現在還沒有 GitHub 成癮 XDD）。或者是我在唸書的時候把手機藏在桌子底下之類的。

至於第二個，書裡有提到「把例行公事和要建立的新習慣綁起來」，我現在就有做例如在吃早餐的時候看新聞，自然就會讓自己關心時事了。

渴望

這邊我有印象的點比較少，比較有印象的是把想要的行為跟需要的行為做配對，例如在滑手機以前做幾下伏地挺身。之後或許可以嘗試看看。

回應

回應就是你做的事。有提到一些覺得還不錯的：

  • 消除/增加你做事情的阻力，會讓習慣容易形成/難以形成。
  • 想要戒掉拖延，可以嘗試兩分鐘法則：新的習慣只需要先做兩分鐘就好，或者見好就收（就是在覺得疲乏以前就停），久了做的時間自然就會拉長了。

這裡對我最有啟發的應該就是兩分鐘法則了。我以前建立新習慣都會讓第一天做好幾個小時，做到很累，隔天就不會想繼續做了。這個法則可以讓心理在建立習慣的時候壓力比較不會那麼大，也讓我很有效的停止沒有生產力的工作，改善拖延。

獎賞

這邊講的獎賞是指做完習慣以後會有什麼回饋。比較有印象的：

  • 獎勵會讓人想繼續做，懲罰會讓行為被避免。
  • 可以用一個顯而易見的東西幫助自己追蹤習慣，例如在紙上畫滿格子，之後每運動一次就塗一格。
  • 為習慣簽訂一個契約可以增加違反習慣時的代價，也可以找一個朋友來幫你監督習慣。

第二個我記得我小時候有做過類似的事，就是在生字簿每寫完一行以後就塗一格。我最近也有嘗試在每次運動以後就塗格子，不過感覺效果不大 QQ。
至於監督習慣的部份，像前幾天我被發現沒有持續發文更新這個 blog，我就在 Discord 上請別人建 Thread 來監督我，我就很任命去做了 XDD。

總之我認為這是一本很棒的書，花幾百塊買他完全值得，希望大家都能參考看看 OwO
