反平方根快速演算法

其實我很久以前就知道這個演算法了，但是那時候看不太懂。直到最近因為計組在教浮點數運算才把他再拿出來看，應該有比之前更了解一點。比較詳細的說明可以參見 維基百科。

這個演算法可以不用函式庫快速的算出 $\frac{1}{\sqrt{x}}$ 的近似值。用 C++ 實作如下（取自維基百科）：

這個演算法主要分成兩步驟：

  1. 巧妙運用浮點數在電腦儲存的方式，求出 $\frac{1}{\sqrt{x}}$ 的一個近似值（第 9-10 行）
  2. 用牛頓法求出更接近的值。（第 11 行）

這裡主要會講解第一個步驟（y = 0x5f3759df - (number >> 1)），第二個步驟的牛頓法，網路上應該滿多講解的，就節省篇幅不講了。

要了解第一個步驟，首先要先了解浮點數的儲存方式。浮點數在電腦儲存的標準由 IEEE 754 制定，可以分為 sign, exponent, mantissa 三個部份。以單精度（也就是 C/C++ 的 float）來說，三個位置佔的 bit 數分別為 $1,8,23$。假如三個位置代表的無號整數分別為 $s,e,m$，那他代表的浮點數就是 $(-1)^s \cdot 2^{e - B} \cdot (1 + \frac{m}{L}), B = 127, L = 2^{23}$。當然他有例外情形，在此略過。（圖取自維基百科）

以這個例子來說，因為開平方根一定要是正數，所以 $s=0$，把他 cast 成整數以後就是 $eL + m$。由 $y = \frac{1}{\sqrt{x}}$ 我們有

\[\log_2(y) = -\frac{1}{2}\log_2(x)\]
\[\log_2(1 + \frac{m_y}{L}) + (e_y-B) = -\frac{1}{2}\log_2(1 + \frac{m_x}{L}) - \frac{1}{2} (e_x-B) \]

這裡需要一個近似值：令 $\log_2(1+a) = a + \sigma$，則有

\[(\sigma + \frac{m_y}{L}) + (e_y-B) = -\frac{1}{2}(\sigma + \frac{m_x}{L}) - \frac{1}{2} (e_x-B) \]

\[m_y + e_y L = -\frac{3}{2}\sigma (B-L) - \frac{1}{2} (m_x + e_x L) \]

$\frac{1}{2} (m_x + e_x L)$ 對應到就是 (number >> 1) 的部份，而 $-\frac{3}{2}\sigma (B-L)$ 在選取適當的 $\sigma$ 以後就是 0x5f3759df。

第一次知道這個演算法覺得滿神奇的，現在也是 owo。只是我覺得這種 magic number 在開發當中應該要適當注釋，不然再一般開發之下明顯不合格。
